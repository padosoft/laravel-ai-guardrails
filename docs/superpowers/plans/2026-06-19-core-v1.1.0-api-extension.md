# Core v1.1.0 — Admin-enabling API extension — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Additively extend the `padosoft/laravel-ai-guardrails` HTTP API (`ai-guardrails.api.v1`) so the admin panel can render pixel-faithfully, then release `v1.1.0`.

**Architecture:** Every change is additive and backward-compatible — new response fields/buckets only, existing shapes untouched, new toggles default-safe. Controllers in `src/Http/*Controller.php` gain fields; the output-stats store gains a nullable `detector` dimension; the settings overridable allow-list widens with validation. No breaking change to any existing key, route, name, or envelope.

**Tech Stack:** PHP ^8.3, Laravel 13, PHPUnit 12 + Testbench 11, Infection (PHAR) ≥80% MSI, Pint, PHPStan level 8.

## Global Constraints

- PHP `^8.3`; tests on PHP 8.5 via Herd: `"%USERPROFILE%\.config\herd\bin\php85.bat" vendor/bin/phpunit`.
- TDD always: failing test → run (FAIL) → minimal impl → run (PASS) → both-states test → mutation gate (`infection --min-msi=80`) → commit.
- Append-only stores never UPDATE/DELETE in place; the model throws on update/delete.
- Every behaviour-changing toggle default-OFF where it changes behaviour, tested in BOTH states (three for enforce/monitor/off).
- Envelope is `{schema_version: "ai-guardrails.api.v1", schema, data}`; route names `ai-guardrails.api.*`; API default-OFF behind `api.enabled`.
- Docs must match code (R9): every new config key/field quoted in README/docs-site must exist. Update `docs-site/docs/**` in the SAME unit of work for any user-facing change, then `cd docs-site && npm run check && npm run build` green.
- One branch per macro task off `main`: `feature/core-v1.1.0-api`. Each task = its own commit; the macro PR goes into `main` with the Copilot review loop green.
- Backward-compat lock: add a test asserting the pre-1.1.0 fields are still present with identical types.
