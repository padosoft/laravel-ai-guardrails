# v1.0.0 — Documented-limitations closeout (design)

Date: 2026-06-18 · Target: tag `v1.0.0` after all five components merge.

Closes every entry in the README "Known limitations" list plus the deferred MCP surface. Each component is a default-safe config toggle, tested in both states, shipped as its own macro-branch/PR with the full DoD loop (phpunit → pint → phpstan → local review → CI incl. Infection ≥80% MSI → merge). Compose-not-couple is preserved and extended in the architecture test.

## L1 — Confusables fold (Control B) — the real security gap

**Problem:** NFKC folds fullwidth/compatibility characters but NOT cross-script homoglyphs. `ignоre` with a Cyrillic `о` (U+043E) evades the screening regexes.

**Design:**
- New `src/Screening/ConfusablesFolder.php` — a curated, maintained map of common cross-script Latin look-alikes → ASCII skeleton: Cyrillic (`а е о р с х у і ј ѕ`…), Greek (`ο ν α ρ`…), and common digit/punctuation confusables. NOT the full Unicode confusables.txt (curated high-value subset, no external data file, avoids over-folding legitimate non-Latin prompts).
- `UnicodePromptNormalizer` gains a `foldConfusables` pass (new trailing ctor bool, default true for the provider; back-compat default in direct construction kept consistent with the other passes). Applied AFTER NFKC + zero-width strip, BEFORE casefold/pattern matching.
- Config `normalization.fold_confusables` (default **on**). Provider wires it.
- It is a one-way skeleton fold for matching only (the stored/audited prompt is unaffected — folding happens in the screener's match path, like the rest of normalization). Documented as skeleton-fold, not reversible normalization.

**Tests:** Cyrillic/Greek homoglyph evasion now blocked; legitimate non-Latin text not mangled into a false positive where avoidable; pass individually toggleable (both-states); folder unit test per script.

## L2 — Robust HTML sanitization (Control C) — compose HTMLPurifier

**Problem:** the built-in `allowlist` mode is convenience-grade, not a real HTML sanitizer.

**Design:**
- New `src/Output/PurifierHtmlSanitizer.php` composing `ezyang/htmlpurifier` (added as `suggest`, optional). Implements the same `OutputSanitizer` (+ `ReportingOutputSanitizer`) contract as `HtmlMarkdownSanitizer` so the middleware/stat path is unchanged.
- `src/Output/HtmlSanitizerFactory.php` keeps the vendor reference inside `src/Output` (compose-not-couple). When `output_handler.html_mode = allowlist`: use HTMLPurifier when `class_exists(\HTMLPurifier::class)`, else **graceful fallback** to the existing built-in allowlist sanitizer. `escape` mode and markdown defanging are unchanged.
- Provider binds the sanitizer via the factory.

**Tests:** with HTMLPurifier present → robust sanitization of a hostile fragment; absent → falls back to the built-in allowlist (both-states). Markdown defang + escape mode unaffected. Reporting (stats/events) still fires.

## L3 — toolCalls defense-in-depth (Control C) — opt-in, default OFF

**Problem (by design):** Control C rewrites `text`/structured only; `toolCalls` are governed by A/D. Force-sanitizing tool args by default would corrupt legitimate tool execution.

**Design:**
- Config `output_handler.sanitize_tool_calls` (default **false** → zero behaviour change; tools never break).
- When ON, `GuardrailOutputMiddleware` also walks the string leaves of each `toolCalls` entry's arguments and applies the SAME sanitize→redact pass (for hosts that render/log tool-call args). Recorded via the existing stat/event path (a new `OutputStatKind`? no — reuse existing kinds; the event already carries kinds). Documented clearly that A/D remain the primary governors and this is an extra opt-in layer that may alter args.

**Tests:** default off → tool args untouched; on → string leaves sanitized; nested args handled; both-states.

## L4 — HITL turnkey (Control D) — verifiable setup

**Problem:** Control D's flow persistence (tokens, resume) is configured by the host.

**Design:**
- `ai-guardrails:hitl-install` artisan command — publishes/runs the `laravel-flow` migrations and writes the recommended `hitl` config, guarded by `class_exists` (clear message when flow absent). Idempotent.
- `ai-guardrails:hitl-status` — diagnostic: reports whether flow is installed, migrated, persistence enabled, and HITL active; non-zero exit when misconfigured.
- README "HITL setup" section.

**Tests:** status command reports correctly with flow present/migrated vs absent; install command guards on flow absence (both-states). No new runtime coupling outside `src/Hitl`/console.

## L5 — MCP surface (4th surface) — laravel/mcp

**Problem:** MCP was a deferred follow-up.

**Design (laravel/mcp v0.8, optional/default-OFF):**
- `src/Mcp/GuardrailsMcpServer.php` extends `Laravel\Mcp\Server` with `$tools`.
- Tools (extend `Laravel\Mcp\Server\Tool`, `schema(JsonSchema)` + `handle(Request): Response`): `ScreenPromptTool` (screen + verdict), `SanitizeOutputTool` (sanitize+redact a blob), `RecentAuditTool` (recent audit attempts, read-only).
- Config block `mcp.enabled` (default **false**, like `api.enabled`). When enabled + `class_exists(\Laravel\Mcp\Server::class)`, the provider registers the server via `Mcp::local('ai-guardrails', GuardrailsMcpServer::class)` in boot (after the open-surface guard). `laravel/mcp` is `suggest`.
- compose-not-couple: the `Laravel\Mcp\*` vendor reference lives ONLY in `src/Mcp`; the architecture test's allowed-vendor map is extended (`src/Mcp → laravel/mcp`, `src/Output → pii + htmlpurifier`, `src/Hitl → flow`).

**Tests:** each tool's handle() returns the expected Response shape (screen blocks an injection, sanitize neutralises HTML, audit lists rows); server registration gated by `mcp.enabled` + class_exists (both-states); arch test still green.

## Cross-cutting

- Every toggle tested in both states; audit stores remain append-only; Infection ≥80% MSI stays green (new pure-logic classes — folder, purifier wrapper — are in scope; MCP/console adapters follow the existing infrastructure-exclusion rationale).
- README + `config/ai-guardrails.php` + PROGRESS updated per component (R9: every quoted key/command/class exists).
- Order: L1 → L2 → L3 → L4 → L5, each its own PR. After all merge + main CI green → tag `v1.0.0` (annotated) + GitHub Release.

## Out of scope (documented, intentional)

- Full Unicode confusables.txt (curated subset chosen).
- Making `toolCalls` sanitization a default (kept opt-in by design).
- A standalone MCP transport beyond `Mcp::local` (host mounts/serves).
